# 0.0.1-dev-21 — compound assignment

_Released 2026-05-26_

`+=`, `-=`, `*=`, `/=` join the assignment family. Cuts a
huge amount of `i = i + 1` noise out of loops, accumulator
patterns, and arithmetic-heavy code.

## Highlights

```mint
import { println } from "mint:io";

export function main(): void {
  let n: int = 10;
  n += 5;
  n -= 3;
  n *= 2;
  n /= 4;
  if (n == 6) { println("ok"); }

  let acc: int = 0;
  for (let i: int = 1; i <= 5; i = i + 1) {
    acc += i;
  }
  if (acc == 15) { println("sum=15"); }
}
```

Rules:

- Both operands must be numeric (`int`, `int64`, `float`,
  `double`); `DLM4077` otherwise.
- LHS still has to be a mutable assignment target — same
  rule as plain `=`.
- Resulting expression type is the LHS type, so `+=` is
  itself an expression (`while ((n += 1) < 10)` etc.).

Lowered to the matching C++ compound operator (`n += 5;`).
No two-evaluation gotcha because the C++ compiler handles
side effects on the LHS itself.

  `PLUS_EQUAL`, `MINUS_EQUAL`, `STAR_EQUAL`,
  `SLASH_EQUAL` tokens.

- `src/parser/parser.ts` — `assignment()` matches the new
  tokens and threads `operator` through.
- `src/semantic/checker.ts` — compound paths require
  numeric operands (`DLM4077`).
- `src/emitter/cppEmitter.ts` — emits the operator
  verbatim.
- `tests/integration/compoundAssignBuild.test.ts` — happy
  path + non-numeric-operand diagnostic.
