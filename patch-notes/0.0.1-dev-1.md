# 0.0.1-dev-1 — Phase 1: language ergonomics

_Released 2026-05-26_

The first bump under the new versioning scheme. Five long-missing
syntactic pieces land in one shot so they stand or fall together.

## Highlights

- **Short-circuit logical operators.** `&&` and `||` are now real
  tokens and have their own precedence layer between ternary and
  equality. Expressions like `a > 0 && b < 100` and
  `name == "" || valid` finally parse.
- **Ternary `cond ? a : b`.** New `ConditionalExpression` AST node;
  branches must share a type and the condition must be bool.
- **String concatenation with `+`.** When both sides are strings the
  checker tags the binary expression and the emitter wraps each
  side in `std::string(...)` so `std::string_view` operands also
  work.
- **Multi-statement lambda bodies.** `fn(p: T): R => { ...; }` now
  parses a real block. The emitter generates
  `[=](params) -> T { ...stmts... }`; single-expression lambdas
  continue to emit the existing `{ return expr; }` form.
- **String template literals.** `"hello ${name}, count=${n}"`
  splits at parse time into a `TemplateLiteralExpression` whose
  parts are stitched together at emit via
  `__doublemint_template_to_string`. Only bare identifiers are
  allowed inside `${}` for now; arbitrary expressions are a future
  bump.

## Files touched

- `src/lexer/scanner.ts`, `src/lexer/token.ts` — `&&` / `||` tokens.
- `src/parser/parser.ts`, `src/parser/ast.ts` — new precedence
  layers, `ConditionalExpression`, `TemplateLiteralExpression`,
  lambda `blockBody`, `splitTemplate` helper.
- `src/semantic/checker.ts` — type rules for `&&`/`||`, ternary,
  template literals, string concat tagging.
- `src/emitter/cppEmitter.ts` — emission for all five features +
  `__doublemint_template_to_string` injected per-module behind an
  include guard so multi-file builds don't redefine it.
- `tests/integration/languageFeaturesBuild.test.ts` — five new
  end-to-end tests.

## Tests / examples

- 141 / 141 tests pass.
- 20 / 20 examples build clean.

## Tooling

- New `pnpm version:bump "<title>"` script that drives the
  `0.0.1-dev-<n>` counter and scaffolds the next patch note.
- `patch-notes/README.md` documents the versioning policy and the
  remaining roadmap to `0.0.1`.
