# 0.0.1-dev-25 — `async` functions and `await`

_Released 2026-05-26_

First Phase 5 brick that introduces a new runtime concept
rather than a new piece of language sugar: declare a
function `async function`, call it, and `await` the result.
Internally each async function returns `std::future<T>`
populated by `std::async(std::launch::async, …)`, and
`await` is a thin wrapper over `.get()`.

## Highlights

```mint
import { println } from "mint:io";

async function fetchValue(): int {
  return 42;
}

async function twice(n: int): int {
  return n * 2;
}

export function main(): void {
  let v: int = await fetchValue();
  if (v == 42) { println("got 42"); }

  let d: int = await twice(21);
  if (d == 42) { println("twice works"); }
}
```

Semantics:

- The declared return type is the *inner* type that the
  body returns. The compiler synthesises the outer
  `Future<T>` wrapper for callers.
- Calling an async function (without `await`) yields a
  `Future<T>` you can hand off, store in a variable, etc.
- `await expr` requires `expr` to be `Future<T>` (`DLM4079`
  otherwise) and unwraps to `T` by blocking on
  `std::future::get()`.

Lowering:

```cpp
std::future<int> fetchValue() {
  return std::async(std::launch::async, [=]() -> int {
    return 42;
  });
}

// caller:
int v = (fetchValue()).get();
```

The `std::launch::async` policy is hard-coded for this
release — every async call spawns a worker thread.
Lazy / `deferred` policies and a pluggable executor can
come later.

## Files touched

- `src/lexer/token.ts`, `src/lexer/scanner.ts` — new
  `ASYNC` / `AWAIT` keywords.
- `src/parser/ast.ts` — `FunctionDeclaration.async`,
  `AwaitExpression` AST node.
- `src/parser/parser.ts` — `async function` declaration
  branch (`DLM2087`), `await` parsed in the unary slot.
- `src/semantic/checker.ts` — async functions register
  with a wrapped `Future<T>` return type;
  `AwaitExpression` validates and unwraps; `Future` added
  to `builtInTypes`.
- `src/emitter/cppEmitter.ts` — `<future>` baked into the
  header preamble + `<future>` source include guarded by
  `moduleUsesAsync`; async functions emit
  `std::future<T>` + `std::async` body; `Future<T>`
  generic emits as `std::future<T>`; `AwaitExpression`
  lowers to `(expr).get()`; walker cases for
  `AwaitExpression` in `collectAssignedRootsFromExpression`
  and `expressionUsesPrint`.
- `tests/integration/asyncAwaitBuild.test.ts` — happy path
  (two async fns) + non-future-await diagnostic.
