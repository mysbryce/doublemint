# 0.0.1-dev-9 — SQLite vendor + `mint:sql`

_Released 2026-05-26_

Mint binaries can now talk to SQLite databases out of the box.
The SQLite amalgamation (sqlite3.c + sqlite3.h) is vendored in
tree and compiled alongside user code, so there is still no
external runtime dependency.

## Highlights

```mint
import { Database, SqlResult } from "mint:sql";

let db: Database = new Database();
db.openMemory();
db.exec("CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT, age INTEGER)");
db.execParams("INSERT INTO users (name, age) VALUES (?, ?)", ["alice", "30"]);
db.execParams("INSERT INTO users (name, age) VALUES (?, ?)", ["bob", "25"]);

let rows: SqlResult = db.query("SELECT id, name, age FROM users ORDER BY age");
while (rows.hasNext()) {
  println(rows.getInt("id"), " ", rows.getString("name"), " age=", rows.getInt("age"));
  rows.next();
}
rows.close();

let bob: SqlResult = db.queryParams("SELECT name, age FROM users WHERE name = ?", ["bob"]);
while (bob.hasNext()) {
  println("found ", bob.getString("name"), " age=", bob.getInt("age"));
  bob.next();
}
bob.close();

db.close();
```

### `Database` class

| Method | Purpose |
| --- | --- |
| `open(path)` / `openMemory()` | open a file or ":memory:" database |
| `close()` | close + release the handle |
| `exec(sql)` | run a non-result statement |
| `execParams(sql, params)` | parameterised execute (`?` bind text) |
| `query(sql)` / `queryParams(sql, params)` | prepare + step and return a `SqlResult` |
| `lastInsertRowId()` | `int64` of the last inserted rowid |
| `changes()` | number of rows affected by the last statement |
| `error` (property) | last error message |

### `SqlResult` class

| Method | Purpose |
| --- | --- |
| `hasNext()` | true while a row is available |
| `next()` | advance the cursor |
| `getString(col)` / `getInt(col)` / `getInt64(col)` / `getDouble(col)` | typed column access by name |
| `isNull(col)` | NULL check |
| `columnCount()` / `columnName(i)` | introspection |
| `error` (property) | last error message |
| `close()` | finalise the statement |

Bind values use SQLite's text affinity (`sqlite3_bind_text`)
since Mint funnels through `string[]` for `*Params`. Numeric
columns still read back correctly via `getInt`/`getDouble`
thanks to SQLite's dynamic typing.

## Implementation

- Vendored SQLite 3.38.2 amalgamation
  (`src/runtime/vendor/sqlite/sqlite3.c` + `.h`, ~8.5 MB
  source). Compiled as a single C TU via the two-pass pipeline.
- `mint:sql`'s `builtinNative` bundle adds the SQLite source +
  vendor include dir and disables threading / load-extension /
  memstatus to keep the build self-contained:
  `SQLITE_THREADSAFE=0`, `SQLITE_OMIT_LOAD_EXTENSION=1`,
  `SQLITE_DEFAULT_MEMSTATUS=0`.
- `src/runtime/headers/sql.hpp` declares `Database` and
  `SqlResult` as shared-pointer-backed value types.
- `src/runtime/sources/sql.cpp` implements both classes and
  internal `DatabaseHolder` / `ResultHolder` structs that hold
  the raw `sqlite3*` / `sqlite3_stmt*` and own their
  cleanup in destructors.
- Mint manifest registers the two classes on `mint:sql`; param
  vectors take `std::vector<std::string_view>` so Mint's
  literal-string-array emitter (dev-3) feeds them directly.
