# 0.0.1-dev-40 — streaming process pipes

_Released 2026-05-27_

Stream child-process stdout (or stdin) line by line via
`popen` — no more buffering the whole output before
inspecting it.

## Highlights

```mint
import { println } from "mint:io";
import { Process } from "mint:process";
import { String } from "mint:string";

export function main(): void {
  let h: int = Process.streamOpen("ping -n 5 localhost", "r");
  if (h < 0) { return; }
  while (!Process.streamEof(h)) {
    let line: string = Process.streamReadLine(h);
    if (String.length(line) > 0) {
      println("got: " + line);
    }
  }
  Process.streamClose(h);
}
```

## API

| Symbol | Signature | Purpose |
| --- | --- | --- |
| `Process.streamOpen` | `(command: string, mode: string): int` | Open a pipe. `"r"` reads child stdout; `"w"` writes child stdin. Returns a handle, or `-1` on failure. |
| `Process.streamReadLine` | `(handle: int): string` | Read until newline. Returns the line without the trailing `\n`; empty string on EOF. |
| `Process.streamEof` | `(handle: int): bool` | True once the child closes its pipe. |
| `Process.streamWriteLine` | `(handle: int, text: string): bool` | Write a line (auto-appends `\n`) + `fflush`. Returns false on write error. |
| `Process.streamClose` | `(handle: int): int` | Closes the pipe and returns the child's exit status (passed through `_pclose` / `pclose`). |

## Implementation

`popen` on POSIX, `_popen` on Windows. Stream handles live
in a tiny `std::vector<std::unique_ptr<StreamHandle>>`
registry inside `doublemint_process_stream`. Closing a
handle frees the `FILE*` but leaves the slot in place
(handles never shift, so user-side indexes stay valid).
