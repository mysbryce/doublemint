# 0.0.1-dev-12 — `doublemint version` CLI

_Released 2026-05-26_

Small ergonomic bump: the CLI now reports the running
toolchain version through three equivalent surfaces, and the
`--help` banner shows it too.

## Highlights

```
$ doublemint version
doublemint 0.0.1-dev-12

$ doublemint --version
doublemint 0.0.1-dev-12

$ doublemint -v
doublemint 0.0.1-dev-12

$ doublemint --help
doublemint 0.0.1-dev-12

Usage:
  doublemint check <entry.dlm>
  doublemint check --stdin-filepath <entry.dlm>
  doublemint emit <entry.dlm>
  doublemint build <entry.dlm> --out <binary> [--compiler <clang++|g++>] [--cpp-out <dir>]
  doublemint version
  doublemint --help
```

`readVersion()` walks up to three candidate `package.json`
locations relative to the running module (`dist/cli.js` vs.
`src/cli.ts` paths), picks the one whose `name` is
`"doublemint"`, and returns its `version`. Falls back to
`0.0.0-unknown` if none is found rather than crashing.

## Files touched

- `src/cli.ts` — adds `readVersion()`; teaches the parser
  about `--version` / `-v` as both a command-level flag and
  a positional `version` command; adds the `version` branch
  to `main()`; prepends the version to the `--help` banner.
