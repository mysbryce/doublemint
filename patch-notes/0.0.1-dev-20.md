# 0.0.1-dev-20 ‚Äî unary `-` and `!`

_Released 2026-05-26_

Small but load-bearing: prefix `-` for numeric negation and
prefix `!` for logical negation, both wired into the unary
precedence slot above `*`/`/`.

## Highlights

```mint
import { println } from "mint:io";

export function main(): void {
  let n: int = -5;
  let m: int = -n;
  if (m == 5) { println("neg-of-neg"); }

  let flag: bool = true;
  if (!flag) { println("never"); } else { println("ok"); }
  if (!(1 == 2)) { println("not-equal"); }
}
```

Type rules:

- `-x` requires a numeric operand (`int`, `int64`, `float`,
  `double`) ‚Äî `DLM4075` otherwise ‚Äî and returns the same
  numeric type.
- `!x` requires a `bool` operand ‚Äî `DLM4076` otherwise ‚Äî and
  returns `bool`.
- Both compose: `-(a + b)`, `!(x == y)`, `--n` are all
  legal.

Lowered to plain C++ `(-(arg))` / `(!(arg))`; the inner
paren keeps `!(1 == 2)` from being read as `(!1) == 2`
under `-Wlogical-not-parentheses`.
