# 0.0.1-dev-10 — `OS.runOutput` / `runExitCode` + args helpers

_Released 2026-05-26_

Wraps Phase 3 with a small but practical extension to `mint:os`:
distinct output / exit-code capture and shell-safe argv
quoting helpers. Full streaming process pipes (`Process.spawn`
with stdin/stdout handles) is parked for a later bump — the
ergonomic gap that mattered most was being unable to read an
exit code separately from stdout.

## Highlights

```mint
import { OS } from "mint:os";

let output: string = OS.runOutput("git rev-parse HEAD");
let code: int = OS.runExitCode("test -f /etc/hosts");

let quoted: string = OS.argsQuote("hello world");
let cmd: string = OS.argsJoin(["node", "script.js", "arg with space"]);
let result: string = OS.runOutput(cmd);
```

| Function | Notes |
| --- | --- |
| `OS.runOutput(cmd)` | combined stdout+stderr; child run via `popen("…2>&1")` |
| `OS.runExitCode(cmd)` | child exit code (-1 if `popen` itself failed) |
| `OS.argsQuote(arg)` | safe quoting (`""` when arg contains spaces / quotes / tabs) |
| `OS.argsJoin(args)` | space-joins after running each piece through `argsQuote` |
| `OS.execute(cmd)` | unchanged single-call helper from earlier bumps |

## Files touched
