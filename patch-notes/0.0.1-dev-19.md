# 0.0.1-dev-19 — `match` expressions

_Released 2026-05-26_

Match grows a value-returning form. Drop the same syntax on
the right-hand side of `let`/`return`/any expression slot
and each arm contributes a value rather than running a
block of statements.

## Highlights

```mint
import { println } from "mint:io";

enum Day { Mon, Tue, Wed }

export function name(d: Day): string {
  return match (d) {
    Day.Mon => "monday",
    Day.Tue => "tuesday",
    _ => "wednesday"
  };
}

export function size(n: int): string {
  let label: string = match (n) {
    0 => "zero",
    1 => "one",
    _ => "many"
  };
  return label;
}
```

Output:

```
monday
wednesday
zero
one
many
```

Rules:

- Each arm is `pattern => expression` (no braces, no
  semicolon — the comma separates arms).
- Every arm's value must have the same type (`DLM4073`).
- A wildcard arm `_ => ...` is required so the expression
  is total (`DLM4074`); duplicate wildcards rejected with
  `DLM4072`.
- Patterns are type-checked against the discriminant via
  the existing `assertAssignable`, so cross-type patterns
  still fail at compile time.

Lowering:

```cpp
([&]() {
  const auto __dlm_match_v = <discriminant>;
  return (__dlm_match_v == <pat1>) ? <val1>
       : (__dlm_match_v == <pat2>) ? <val2>
       : <wildcardVal>;
})()
```

The discriminant is captured once in an IIFE so it can be
used as a sub-expression anywhere (initializer,
`return`, function argument, nested match).
