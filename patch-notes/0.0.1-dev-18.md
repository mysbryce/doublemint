# 0.0.1-dev-18 — `match` statements

_Released 2026-05-26_

Pattern matching arrives. Statement-form `match` over any
expression — int literals, strings, enum variants — with a
`_` wildcard for the catch-all arm. Lowered to a hoisted
`const auto __dlm_match_N = …;` plus an `if`/`else if`
ladder, so the discriminant is evaluated exactly once and
the lowering inherits the same equality semantics every
other operator uses.

## Highlights

```mint
import { println } from "mint:io";

enum Day { Mon, Tue, Wed, Fri }

export function classify(n: int): string {
  match (n) {
    0 => { return "zero"; },
    1 => { return "one"; },
    _ => { return "many"; }
  }
  return "unreachable";
}

export function main(): void {
  let d: Day = Day.Wed;
  match (d) {
    Day.Mon => println("monday"),
    Day.Tue => println("tuesday"),
    Day.Wed => {
      println("wednesday");
      println("midweek");
    }
    _ => println("other")
  }
}
```

Arm bodies:

- **Block body** `=> { ... }` — multiple statements, can
  include `return`, decls, nested matches, anything.
- **Expression body** `=> println(...)` — single
  expression-statement, written without semicolon. Trailing
  `,` is required between expression arms; optional after
  block arms.

Diagnostics:

- `DLM2090`-`DLM2096` for the parse layer (missing parens,
  brace, arrow, comma, empty match).
- `DLM4071` when more than one `_` arm appears.
- Pattern expressions are type-checked against the
  discriminant via the existing `assertAssignable`, so a
  `Day` discriminant rejects an `int` pattern at compile
  time.

## Files touched

- `src/lexer/token.ts`, `src/lexer/scanner.ts` — `match`
  keyword (the existing `ARROW` token already covers both
  `=>` and `->`).
- `src/parser/ast.ts` — `MatchStatement`, `MatchArm`,
  `MatchPattern` types.
- `src/parser/parser.ts` — `matchStatement()` /
  `matchPattern()`, hooked into the statement dispatcher.
- `src/semantic/checker.ts` — `MatchStatement` validation,
  pattern type-check, duplicate-wildcard rejection.
- `src/emitter/cppEmitter.ts` — `emitMatchStatement()` plus
  cases in the `collectStringViewDeclarations`,
  `collectAssignedRoots`, `statementUsesPrint`, and
  `statementUsesDefer` recursive walkers.
- `tests/integration/matchBuild.test.ts` — happy-path
  enum/int dispatch plus duplicate-wildcard diagnostic.
