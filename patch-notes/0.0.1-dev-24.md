# 0.0.1-dev-24 — numeric `.toString()`

_Released 2026-05-26_

`n.toString()` works for every numeric type. No import
required — the checker recognises the method on `int`,
`int64`, `float`, `double` and lowers it directly to
`std::to_string(n)`.

## Highlights

```mint
import { println } from "mint:io";

export function main(): void {
  let n: int = 42;
  println("n=" + n.toString());

  let pi: float = 3.14;
  println("pi=" + pi.toString());

  let big: int64 = 1000;
  println("big=" + big.toString());
}
```

Output:

```
n=42
pi=3.140000
big=1000
```

Notes:

- Zero-arg call only — `n.toString(2)` (base) is not
  supported in this bump.
- Reuses the existing `primitiveExtensionNative` slot on
  `MemberExpression`, so the emitter already knows how to
  rewrite the call into `std::to_string(receiver)`.
- Float formatting follows `std::to_string` defaults (six
  digits after the point); a richer formatter can come
  later via `mint:fmt`.
