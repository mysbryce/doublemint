# 0.0.1-dev-6 — SHA-256 + MD5 + HMAC in `mint:crypto`

_Released 2026-05-26_

Closes out the Phase 2 stdlib goals. `mint:crypto` graduates
from FNV1A + XOR helpers into a real (no-OpenSSL, single-file)
cryptographic hash surface that matches every standard test
vector.

## Highlights

Four new entries on the `Crypto` namespace:

| Function | Spec | Verified |
| --- | --- | --- |
| `Crypto.sha256(value)` | FIPS 180-4 | `sha256("") = e3b0…b855`, `sha256("abc") = ba78…15ad` |
| `Crypto.md5(value)` | RFC 1321 | `md5("") = d41d…427e`, `md5("abc") = 9001…7f72` |
| `Crypto.hmacSha256(key, message)` | RFC 2104 | RFC 4231 `("key", "The quick brown fox jumps over the lazy dog") = f7bc…3cd8` |
| `Crypto.hmacMd5(key, message)` | RFC 2104 | RFC 2202 `("key", same) = 8007…e275` |

All hashes return lowercase hex. The existing FNV1A / XOR /
toHex helpers are untouched, so old call-sites keep compiling.

## Files touched

- `src/runtime/sources/crypto.cpp`:
  - `doublemint_crypto_detail::bytesToHex` shared helper.
  - SHA-256 block processor (`sha256Process` /
    `sha256Bytes`) — rotr-based round, 64 constants, 8-word
    state.
  - MD5 block processor (`md5Process` / `md5Bytes`) —
    classic 64-step rotation table.
  - Public wrappers `__doublemint_crypto_sha256` / `_md5` /
    `_hmac_sha256` / `_hmac_md5` (all key-padding to 64 bytes,
    ipad `^0x36` / opad `^0x5c`).
- `src/builtins/mintModules.ts` — extends mint:crypto with
  the four new functions and adds `<cstring>` / `<vector>` to
  the include list.
- `tests/integration/mintExtendedLibrariesBuild.test.ts` —
  new test asserts the exact RFC test vectors for both hashes
  and HMACs.
