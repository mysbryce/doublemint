# 0.0.1-dev-32 — match arm guards

_Released 2026-05-27_

Arm guards land on both forms of `match`. Add an
`if <bool>` between the pattern and the `=>` and the arm
only fires when the guard passes.

## Highlights

```mint
import { println } from "mint:io";

export function classify(n: int): string {
  return match (n) {
    _ if n < 0 => "negative",
    0 => "zero",
    _ if n < 10 => "small",
    _ => "big"
  };
}

export function main(): void {
  println(classify(-3));   // negative
  println(classify(0));    // zero
  println(classify(5));    // small
  println(classify(100));  // big

  let v: int = 7;
  match (v) {
    _ if v < 5 => println("small"),
    _ if v < 10 => println("medium"),
    _ => println("large")
  }
}
```

Rules:

- Guard expression must be `bool` (`DLM4083`).
- Multiple wildcard arms are allowed *as long as each one
  has a guard*; the unguarded-wildcard uniqueness rule
  still applies (`DLM4071` / `DLM4072` reword to "more
  than one *unguarded* wildcard arm").
- Statement form: lowered into the if / else-if ladder as
  `if (discriminant == pattern && guard)` per arm; pure
  wildcards become `else { … }`, guarded wildcards become
  `else if (guard) { … }`.
- Expression form: each arm is `(eq) && (guard) ? value :`
  in the chained ternary lowering; the final unguarded
  wildcard sits at the tail.

The hoisted discriminant variable gets `[[maybe_unused]]`
so wildcard-only matches don't trip
`-Werror=unused-variable`.

## Files touched

- `src/parser/ast.ts` — `MatchArm.guard?` and
  `MatchExpressionArm.guard?` slots.
- `src/parser/parser.ts` — `matchStatement()` /
  `matchExpression()` accept an optional `if <expression>`
  between pattern and `=>`.
- `src/semantic/checker.ts` — guard expressions are
  type-checked to `bool` (`DLM4083`); wildcard-uniqueness
  diagnostics now read "more than one unguarded
  wildcard".
- `src/emitter/cppEmitter.ts` — `emitMatchStatement()` and
  `emitMatchExpression()` fold the guard into each arm's
  condition; both temporaries gain `[[maybe_unused]]`.

  + non-bool guard diagnostic.
- `tests/integration/matchBuild.test.ts` — assertion
  updated for the new diagnostic phrasing.
