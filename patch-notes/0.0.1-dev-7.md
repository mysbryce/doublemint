# 0.0.1-dev-7 — WebSocket on uWebSockets

_Released 2026-05-26_

First Phase 3 bump. The uWebSockets backend that landed for HTTP
now exposes its WebSocket surface as a first-class Mint API, so
the same `Http` instance that serves HTTP routes can also run
duplex sockets without an external library.

## Highlights

```mint
import { Http, Context, WebSocket } from "mint:http";

let app: Http = new Http();

app.get("/", fn(ctx: Context): void =>
  ctx.html("<h1>Demo</h1><p>Connect to ws://host/ws</p>")
);

app.ws("/ws",
  fn(ws: WebSocket): void => {
    println("connected from ${ws.remoteAddress()}");
    ws.send("welcome");
  },
  fn(ws: WebSocket, msg: string): void => ws.send("echo: " + msg),
  fn(ws: WebSocket): void => println("client disconnected")
);

app.listen("127.0.0.1", 3002);
```

`Http.ws(pattern, onOpen, onMessage, onClose)` accepts three
Mint lambdas. The wrapper translates each to a uWS
`WebSocketBehavior<WsPerSocket>` field. `WebSocket` itself is a
new class wrapping the underlying `uWS::WebSocket<false, true,
WsPerSocket>*` via a `void*` so Mint user code never sees uWS
template noise.

`WebSocket` methods:

| Method | Notes |
| --- | --- |
| `ws.send(text)` | sends as `OpCode::TEXT` |
| `ws.sendBinary(int[])` | sends as `OpCode::BINARY` |
| `ws.close()` | best-effort close |
| `ws.closeWithCode(code, reason)` | RFC 6455 close frame |
| `ws.remoteAddress()` | textual remote endpoint |

Live-verified with `System.Net.WebSockets.ClientWebSocket`
against a Mint binary: connect, receive welcome, send a
message, get the echo, clean close.

## Files touched

- `src/runtime/headers/http.hpp` — new `WebSocket` class
  (opaque-pointer wrapper) and `Http::ws` declaration.
- `src/runtime/sources/http.cpp` — `WsPerSocket` user-data
  struct, `WebSocket` method bodies, `Http::ws` registering
  the `WebSocketBehavior<WsPerSocket>` with open/message/close
  bridges.
- `src/builtins/mintModules.ts`:
  - Registers `WebSocket` class with its 5 methods.
  - Extends `Http` with the `ws(pattern, open, message,
    close)` method.
- `examples/builtins/websocket/main.dlm` — full demo server
  with HTML landing page + WS endpoint.
- `tests/integration/mintWebSocketBuild.test.ts` — compile-only
  smoke test (uWS templates instantiate, binary lands on disk).
