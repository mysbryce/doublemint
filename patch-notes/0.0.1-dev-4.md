# 0.0.1-dev-4 — `mint:array` + generic free functions

_Released 2026-05-26_

`mint:array` ships, but the actually-novel piece is the
language-level change underneath: namespace functions can now
declare generic type parameters (`T`, `U`, …) that the checker
infers from the call site. Without this, every
map/filter/reduce would have to be hand-rolled per element
type.

## Highlights

### Type inference for generic free functions

Any single-uppercase-letter identifier (`T`, `U`, `K`, `V`, …)
in a builtin function signature is now treated as a placeholder.
`validateCallArguments` collects substitutions from each
argument (walks into `ArrayType` / `FunctionType` /
`GenericType` / `OptionalType` recursively) and applies them
to the return type before handing the inferred type back to
the caller.

```mint
let doubled: int[] = Array.map(xs, fn(x: int): int => x * 2);
// declared: map(T[], (T)=>U) -> U[]
// inferred: T=int, U=int -> returns int[]
```

### `mint:array`

Generic-friendly array operations exposed under the `Array`
namespace:

| Function | Notes |
| --- | --- |
| `map(xs, fn)` | `fn: (T) => U`, returns `U[]` |
| `filter(xs, fn)` | predicate of `(T) => bool`, returns `T[]` |
| `reduce(xs, seed, fn)` | seed sets `U`, fn `(U,T) => U` |
| `findIndex(xs, fn)` | returns `int`, -1 on miss |
| `any(xs, fn)` / `all(xs, fn)` | shortcut predicates |
| `reverse(xs)` / `sort(xs)` / `sortBy(xs, cmp)` | structural ops |
| `length(xs)` | `int` |
| `concat(a, b)` / `slice(xs, start, end)` | combinators |
| `contains(xs, x)` / `indexOf(xs, x)` | lookup |

All of these are also automatically callable as methods on
array values thanks to the primitive-extension
generalisation done in this bump (see below):

```mint
xs.map(fn(x: int): int => x * 2)
xs.filter(fn(x: int): bool => x > 0)
xs.reduce(0, fn(acc: int, x: int): int => acc + x)
xs.sort()
```

### Extension-method generalisation

`resolvePrimitiveExtension` (introduced in dev-3) no longer
hard-codes the primitive set. It now matches any receiver whose
type unifies with the namespace function's first parameter via
the same generic-substitution machinery, so `int[].map(...)`
works for free.

## Files touched

- `src/runtime/sources/array.cpp` — 14 templated helpers
  (`map` / `filter` / `reduce` / `findIndex` / `any` / `all` /
  `reverse` / `sort` / `sortBy` / `length` / `concat` /
  `slice` / `contains` / `indexOf`).
- `src/builtins/mintModules.ts` — registers `mint:array` with
  the `Array` namespace + `T`/`U` placeholders in declarations.
- `src/emitter/cppEmitter.ts` — adds `"mint:array": "array"`
  to the source-module map.
- `src/semantic/checker.ts` —
  - `validateCallArguments` now returns
    `Map<string, TypeNode>` substitutions and walks
    declarations vs actuals via new
    `collectGenericSubstitutions` /
    `applyGenericSubstitution`.
  - Every call-site that returned
    `callee.functionType.returnType` now wraps it in
    `applyGenericSubstitution`.
  - `resolvePrimitiveExtension` now accepts any receiver type
    (not just primitives) and uses the same substitution
    path, enabling `xs.map(...)` style method calls on arrays.
- `tests/integration/mintArrayBuild.test.ts` — two end-to-end
  tests (`map`/`filter`/`reduce` + `findIndex`/`any`/`all`/
  `sort`).
- `examples/builtins/array/main.dlm` — demo program walking
  the whole `Array` surface.
