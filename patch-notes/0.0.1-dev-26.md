# 0.0.1-dev-26 — `bool.toString()`

_Released 2026-05-26_

Closes the gap left by dev-24: every primitive now has a
zero-arg `.toString()`. `true.toString()` is `"true"`,
`false.toString()` is `"false"`, and the lowering avoids
the surprising default `1` / `0` C++ produces when you
stream a `bool`.

## Highlights

```mint
import { println } from "mint:io";

export function main(): void {
  let t: bool = true;
  let f: bool = false;
  println("t=" + t.toString());
  println("f=" + f.toString());
  println("expr=" + (1 == 1).toString());
}
```

Output:

```
t=true
f=false
expr=true
```

## Files touched

- `src/semantic/checker.ts` — extra branch in
  `inferCallType` that recognises `.toString()` on a
  `bool` receiver and routes it through the existing
  `primitiveExtensionNative` slot.
- `src/emitter/cppEmitter.ts` — small
  `__doublemint_bool_to_string` helper sits in the
  header preamble alongside
  `__doublemint_template_to_string`, so the emit cost is
  one inline function shared by every translation unit.
- `tests/integration/boolToStringBuild.test.ts` — verifies
  `true`, `false`, and the parenthesised boolean
  expression case.
