# 0.0.1-dev-16 — `doublemint repl`

_Released 2026-05-26_

Interactive evaluator. Each line is wrapped, compiled to a
real native binary, and executed — so REPL semantics are
*exactly* the language semantics, no interpreter to drift.

## Highlights

```
$ doublemint repl
doublemint repl. type :help for commands, :exit to quit.
>> import { println } from "mint:io";
+ import (1 total)
>> import { String } from "mint:string";
+ import (2 total)
>> let name: string = "world";
+ decl (1 total)
>> println("hello " + name);
hello world
>> println(name.upper());
WORLD
>> :exit
```

Meta commands:

| Command | Purpose |
| --- | --- |
| `:help` | command reference |
| `:show` | dump accumulated imports / decls |
| `:reset` | clear all state |
| `:exit` / `:quit` | leave the repl |

Input rules:

- `import { ... } from "mint:..."` → remembered as imports
  and emitted at the top of every compile.
- `let ` / `const ` → remembered as persistent declarations
  inside `main()`, so later expressions can reference them.
- Anything else → appended as a statement in `main()`.
  Auto-`;` is added when missing; lines ending with `}` are
  left alone for block-style input.

Each evaluation writes a fresh `.dlm` into a per-line temp
dir, runs the full resolver / semantic / emitter / native
pipeline, executes the resulting `.exe`, and pipes its
stdout/stderr to the user. `warningsAsErrors` is forced off
inside the REPL so unused-but-not-yet-referenced bindings
don't trip the build.

## Files touched

- `src/repl/repl.ts` — new event-driven readline loop, line

  wrapper program.
- `src/cli.ts` — registers `repl` command, updates `--help`
  banner, threads the call through `main()`.
