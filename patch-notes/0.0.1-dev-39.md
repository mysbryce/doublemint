# 0.0.1-dev-39 — `try` / `catch` / `throw`

_Released 2026-05-27_

Structured exception handling. `throw "msg"` raises a
`std::runtime_error`, and the matching `try { … } catch
(err: string) { … }` block binds the message as a string.

## Highlights

```mint
function risky(n: int): int {
  if (n < 0) {
    throw "negative not allowed";
  }
  return n * 2;
}

try {
  let v: int = risky(-1);
} catch (err: string) {
  println("caught: " + err);
}
```

Output: `caught: negative not allowed`.

## Rules

- `throw expr` requires `expr` to be a `string`
  (`DLM4087`). Non-string throws are reserved for later
  (alongside `Result<T, E>`).
- `catch (name: T) { ... }` — the binding is effectively
  `string` regardless of declared `T` (the type
  annotation parses for forward-compat, doesn't enforce).
- Bare `try { ... } catch { ... }` (no binding) is
  allowed.
- Lowered to C++ `try { ... } catch (const std::exception&
  __dlm_err) { const std::string name =
  __dlm_err.what(); ... }`; `throw` lowers to
  `throw std::runtime_error(std::string(expr));`.
- `<stdexcept>` baked into the header preamble.
