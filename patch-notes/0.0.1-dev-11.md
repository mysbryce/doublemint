# 0.0.1-dev-11 — `mint:test` framework

_Released 2026-05-26_

Tiny built-in test runner so Mint programs can express
test suites without reaching for an external harness.

## Highlights

```mint
import { Test } from "mint:test";
import { String } from "mint:string";

export function main(): void {
  Test.run("addition", fn(): void => {
    Test.expectInt(1 + 1, 2, "1+1");
    Test.expectInt(10 - 3, 7, "10-3");
  });

  Test.run("string upper", fn(): void => {
    Test.expectString(String.upper("mint"), "MINT", "upper(mint)");
  });

  Test.run("bool", fn(): void => {
    Test.expectBool(true, true, "truth");
    Test.expectTrue(5 > 3, "5>3");
  });

  Test.report();
}
```

Output:

```
  addition ... ok
  string upper ... ok
  bool ... ok

2 passed, 0 failed
```

A failing case prints `FAIL`, accumulates a message, and the
final `Test.report()` returns 1 (so the test binary's exit
code reflects pass/fail for CI integration).

| Function | Notes |
| --- | --- |
| `Test.run(name, fn)` | runs a test; prints name, ok / FAIL |
| `Test.expectTrue(value, label)` | assertion on bool |
| `Test.expectInt(actual, expected, label)` | int equality |
| `Test.expectString(actual, expected, label)` | string equality |
| `Test.expectBool(actual, expected, label)` | bool equality |
| `Test.report()` | prints summary, returns 0 / 1 |
| `Test.passed()` / `Test.failed()` | counts |

Internally the framework keeps a static registry counting
passes/failures and a vector of failure messages. Throw inside
a test body and it's caught + reported as `FAIL (threw)` rather
than aborting the binary.
