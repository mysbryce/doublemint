# 0.0.1-dev-30 — shift `<<` / `>>`

_Released 2026-05-27_

Integer shifts join the operator family. The lexer keeps
`<` and `>` as one-character tokens (so generic-type
parsing stays the same), and a new shift precedence layer
in the parser detects two consecutive `LESS` / `GREATER`
tokens to assemble `<<` / `>>` at expression time.

## Highlights

```mint
import { println } from "mint:io";

export function main(): void {
  if ((1 << 4) == 16) { println("shl ok"); }
  if ((256 >> 2) == 64) { println("shr ok"); }

  let n: int = 1;
  let acc: int = 0;
  for (let i: int = 0; i < 4; i++) {
    acc = acc | (n << i);
  }
  if (acc == 15) { println("loop ok"); }
}
```

Generics still parse the C++20-style way:

```mint
let grid: int[][] = [[1, 2], [3, 4]];
```

(Tracking `Array<Array<int>>` literal generic syntax is
separate work — the language uses the `[][]` shorthand for
nested arrays today.)

## Files touched

- `src/parser/ast.ts` — `BinaryExpression.operator` union
  widened with `"<<" | ">>"`.
- `src/parser/parser.ts` — new `shift()` precedence layer
  between `comparison` and `cast`; `peekNext()` helper for
  the LESS-LESS / GREATER-GREATER lookahead. `comparison`
  now drops into `shift()` (instead of straight to `cast()`).
- `src/semantic/checker.ts` — integer-operand check
  (`DLM4080`) extended to cover `<<` and `>>`.
- `src/emitter/cppEmitter.ts` — shift binaries also get
  paren-wrapped, same rule used for the bitwise binaries.
- `tests/integration/shiftBuild.test.ts` — happy path,
  nested-generic regression guard, float-operand
  diagnostic.
