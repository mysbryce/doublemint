# 0.0.1-dev-17 — `enum` declarations

_Released 2026-05-26_

First Phase 5 brick: unit enums. Lowered to C++
`enum class`, so variant access is namespaced and equality
is strongly typed.

## Highlights

```mint
import { println } from "mint:io";

enum Color {
  Red,
  Green,
  Blue
}

export function describe(c: Color): string {
  if (c == Color.Red) { return "red"; }
  if (c == Color.Green) { return "green"; }
  return "blue";
}

export function main(): void {
  let c: Color = Color.Green;
  println(describe(c));
  if (c != Color.Red) { println("not red"); }
}
```

Output:

```
green
not red
```

Semantics:

- `Color.Red` looks up the variant on the enum symbol that
  was registered in scope; unknown variants raise
  `DLM4070`.
- Duplicate variants are rejected at parse time
  (`DLM2083`), empty enums at parse time (`DLM2086`).
- Enum-typed values compare with `==` / `!=` against other
  variants of the same enum.
- Emitted as `enum class Color { Red, Green, Blue };`; the
  member emitter switches `.` to `::` for `Color.Red` only
  when the receiver is a known enum name.

## Files touched

- `src/lexer/token.ts`, `src/lexer/scanner.ts` — `enum`
  keyword.
- `src/parser/ast.ts` — `EnumDeclaration` AST node.
- `src/parser/parser.ts` — declaration branch + variant
  list parser with `DLM2080`-`DLM2086` diagnostics.
- `src/resolver/moduleGraph.ts` — `EnumDeclaration` in the
  `AstModuleExport.declaration` union and in
  `exportFromDeclaration()`.
- `src/semantic/checker.ts` — `enum` symbol kind, registers
  enums in both `types` and `values` so member access on
  the identifier resolves; new `DLM4070` for unknown
  variants; cross-module imports register on the types side
  too.
- `src/emitter/cppEmitter.ts` — emits `enum class`, threads
  an `enumNames` set through `EmitContext`, switches `.` to
  `::` for enum receivers.
- `tests/integration/enumBuild.test.ts` — end-to-end build
  + unknown-variant diagnostic.
