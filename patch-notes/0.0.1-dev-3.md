# 0.0.1-dev-3 — Primitive extension methods (UFCS-style)

_Released 2026-05-26_

Sugar that makes Mint code read like other modern languages. Any
namespace function whose first parameter matches a primitive type
is automatically callable as a method on that primitive.

## Highlights

```mint
import { String } from "mint:string";

let name: string = "doublemint";

// All of these now work as method calls:
name.upper();              // -> String.upper(name)
name.length();             // -> String.length(name)
name.contains("mint");     // -> String.contains(name, "mint")
name.substring(6, 10);     // -> String.substring(name, 6, 10)
"  hi  ".trim();           // works on literals too
"a,b,c".split(",");        // returns string[]
```

Resolution rules:

- Only fires when the receiver's type canonically matches one of
  the primitive types (`string`, `int`, `int64`, `float`,
  `double`, `bool`).
- The method name must exist in some imported namespace as a
  function whose first parameter equals the receiver type.
- Remaining parameters are matched against the call's arguments
  with the existing `validateCallArguments` pass.
- Class method resolution still wins when both could apply, so
  `Queue<int>.push(5)` etc. behave exactly as before.

## Files touched

- `src/parser/ast.ts` — `MemberExpression.primitiveExtensionNative`
  optional annotation set by the checker, consumed by the emitter.
- `src/semantic/checker.ts` — new `Scope.allNamespaceSymbols()`
  iterator over every visible namespace, plus a
  `resolvePrimitiveExtension` helper that walks them looking for
  a `(receiver, …)` overload. `inferCallType` now consults this
  after the existing class-method path.
- `src/emitter/cppEmitter.ts` — `CallExpression` on a marked
  `MemberExpression` is emitted as
  `<nativeName>(<receiver>, <args…>)`.
- `tests/integration/mintStringBuild.test.ts` — new method-style
  end-to-end test confirms `name.upper()` / `"  hi  ".trim()` /
  `"a,b,c".split(",")` all compile and produce identical output to
  the namespace form.
