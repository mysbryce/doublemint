# 0.0.1-dev-37 — `Async.run(fn)`

_Released 2026-05-27_

Convert any zero-arg closure into a `Future<T>` without
declaring a new `async function`.

## Highlights

```mint
import { Async } from "mint:async";

let f: Future<int> = Async.run(fn(): int => 21 * 2);
let v: int = await f;  // 42
```

The signature is generic in `T` — the checker infers `T`
from the lambda's return type and propagates it through
the `Future<T>` return.

## How it works

Signature in the builtin manifest:

```ts
Async.run<T>(fn: function(): T): Future<T>
```

Lowered to a tiny template helper inlined in the header
preamble of every emitted module:

```cpp
template <typename F>
inline auto __doublemint_async_run(F fn) {
  return std::async(std::launch::async, std::move(fn));
}
```

Mint side `Async.run(fn(): int => 21 * 2)` lowers to
`__doublemint_async_run([&]() -> int { return 21 * 2; })`,
which C++ deduces as `std::future<int>` — same as the
async-function form.

## Why this is useful

Async functions are great when the unit of work has a
name. `Async.run` is for the inline case — ad-hoc work
you want to background without declaring a top-level
helper:

```mint
let chunks: Future<int>[] = [
  Async.run(fn(): int => compute(0, 1000)),
  Async.run(fn(): int => compute(1000, 2000)),
  Async.run(fn(): int => compute(2000, 3000))
];

let total: int = 0;
for (let f of chunks) {
  total += await f;
}
```
