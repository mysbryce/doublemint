# 0.0.1-dev-2 — Phase 2 step 1: `mint:string`

_Released 2026-05-26_

First Phase 2 bump. Adds a real string-manipulation module so user
code stops reaching for raw C++ helpers or hand-rolling primitives.

## Highlights

`mint:string` exposes a single `String` namespace with 19 functions:

| Category | Functions |
| --- | --- |
| Case folding | `upper`, `lower` |
| Trimming | `trim` |
| Splitting / joining | `split`, `join` |
| Replace / search | `replace`, `contains`, `startsWith`, `endsWith`, `indexOf`, `lastIndexOf` |
| Slicing | `substring`, `length`, `reverse` |
| Padding / repeating | `repeat`, `padLeft`, `padRight` |
| Number bridge | `fromInt`, `toInt` |

All inputs are `string` (passed as `std::string_view` to the C++
side, zero-copy where possible). `split` returns `string[]` and
pairs with `join` for round-trip CSV-like work. `indexOf` /
`lastIndexOf` return `-1` when the needle is missing.

## Files touched

- `src/runtime/sources/string.cpp` — new runtime, 19 helpers in
  `std::string_view`-first form so unchanged Mint strings don't pay
  for an alloc.
- `src/builtins/mintModules.ts` — registers `mint:string` with a
  `String` namespace.
- `src/emitter/cppEmitter.ts` — adds `"mint:string": "string"` to
  the source-module map so the runtime snippet is pulled in on
  import.
- `examples/builtins/string/main.dlm` — exercises every member of
  `String` against the new template-literal + string-concat syntax.
- `tests/integration/mintStringBuild.test.ts` — two integration
  tests covering casefolding/splitting/search and
  slicing/padding/number bridges.
